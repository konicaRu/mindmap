# Context Mind Map

Интерактивная доска концепций (по образцу
[context-engineering-mindmap](https://safreliy.github.io/context-engineering-mindmap/)):
создавайте карточки, связывайте их, по клику справа открывается контекст-комментарий,
разбитый на настраиваемые секции. Фильтр по категориям, перетаскивание, зум и
панорамирование. Публикуется как статический сайт на GitHub Pages или любом
бесплатном хостинге.

## Возможности

- Карточки-узлы с заголовком, категорией, тегами и источниками.
- **Настраиваемые секции** контекста (Markdown): добавляйте/называйте/удаляйте/сортируйте.
- Связи между карточками (перетаскиванием от края карточки), подписи на связях.
- Фильтр по категориям (вкладки сверху), управление списком категорий.
- Два режима: **Редактирование** и **Просмотр** (read-only).
- Хранение в браузере (localStorage, автосохранение) + **экспорт/импорт JSON**.
- Тёмная/светлая тема.
- Без бэкенда — чистая статика.

## Локальный запуск

```bash
npm install
npm run dev        # http://localhost:5173
```

Сборка статики:

```bash
npm run build      # результат в dist/
npm run preview    # локальный предпросмотр сборки
```

## Как пользоваться

- **Создать карточку:** кнопка «+ Карточка» или двойной клик по холсту.
- **Связать карточки:** в режиме редактирования потяните от правого края одной
  карточки к левому краю другой.
- **Редактировать карточку:** кликните по ней — справа откроется панель.
- **Удалить:** кнопка «Удалить карточку» в панели, либо клавиша Delete на выбранном
  узле/связи.
- **Сохранение:** всё автосохраняется в браузере. Для переноса/публикации —
  «Экспорт» (скачает `.json`) и «Импорт».

## Публикация на GitHub Pages

1. Отредактируйте карту и нажмите **«Экспорт»** — получите JSON-файл.
2. Положите его в репозиторий как `public/data.json` (заменив демо).
   В режиме просмотра приложение грузит именно этот файл, если в браузере зрителя
   ещё нет своих данных.
3. Запушьте в ветку `main`. Workflow [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml)
   соберёт проект и опубликует на Pages. В настройках репозитория включите
   **Settings → Pages → Source: GitHub Actions**.
4. `base` для project-страниц (`/<repo>/`) проставляется автоматически из имени
   репозитория (переменная `BASE_PATH`).

Дайте зрителям ссылку с `?view=1`, чтобы карта открывалась сразу в режиме просмотра, например:
`https://<user>.github.io/<repo>/?view=1`

## Другой хостинг (Netlify / Vercel / Cloudflare Pages)

Команда сборки — `npm run build`, каталог публикации — `dist`. Для корневого
домена `BASE_PATH` не нужен (по умолчанию `/`).

## Модель данных

Доска целиком — один JSON-объект (см. [`src/types.ts`](src/types.ts)):
`Board` → `categories[]`, `nodes[]` (с `sections[]`, `tags[]`, `sources[]`), `edges[]`.
Экспорт/импорт — сериализация этого объекта.
